所有 `/v1/*` API 请求都必须携带服务端 API Key。

```http
Authorization: Bearer <DIMILINKS_API_KEY>
Content-Type: application/json
```

## 示例

```bash
curl "https://dimilinks.com/v1/models" \
  -H "Authorization: Bearer $DIMILINKS_API_KEY"
```

## 安全建议

- 在服务端保存 API Key。
- 不要把完整 API Key 放入浏览器、移动端安装包或公开仓库。
- 如果你需要让终端用户发起图片生成，请由你的服务端代理请求。
- 发现密钥泄露后，请立即禁用旧密钥并创建新密钥。

## 图片结果 URL

图片结果 URL 通常形如 `/p/img/...`。它不需要 Bearer Token，但依赖 URL 自带的过期时间和签名参数。

如果 URL 已过期，请重新查询任务或重新生成图片。
